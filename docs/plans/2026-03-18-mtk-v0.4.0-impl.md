# mtk v0.4.0 Archival Focus Purge: Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove features that overlap with clerk or are redundant with MCP SQL, tightening mtk around its archival mission.

**Architecture:** Each task is a complete, atomic removal of one module and ALL its references across the codebase. Every task leaves the full test suite passing. Tasks are ordered by dependency: data model changes bundled with their dependents, not separated.

**Tech Stack:** Python, SQLAlchemy ORM, SQLite, Typer CLI, pytest

**Spec:** `docs/plans/2026-03-18-mtk-v0.4.0-purge.md`

**Invariant:** After every task, `pytest --tb=short -q` passes. No exceptions.

---

## Task Overview

1. Remove Person/PersonEmail + people module (biggest, touches most files)

6. Remove CLI commands (inbox, show, thread, reply, stats, shell, maildir)

8. Update MCP server, config, pyproject.toml
